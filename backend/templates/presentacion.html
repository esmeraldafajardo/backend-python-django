🚀 Descripción General

Este sistema fue desarrollado con Django y SQLite. Permite gestionar empresas, productos, inventario, autenticación de usuarios con roles y funciones modernas como generación de PDF, simulación de correo y una funcionalidad adicional de IA.

También puedes acceder a esta presentación como una vista HTML en el sistema: /presentacion/

📂 Tecnologías Usadas

  • Django 3 + Django REST Framework
  • SQLite (desarrollo local)
  • ReportLab (para generar PDF)
  • Email backend consola (para pruebas de envío)
  • Postman para pruebas

👤 Usuarios y Autenticación

Se utiliza un modelo personalizado CustomUser con dos roles:

  • admin: acceso total, puede gestionar todo.
  • externo: solo puede consultar empresas y productos.
POST /api/login/
{ "username": "usuario", "password": "contraseña" }

📚 Funcionalidades

  • Empresas: CRUD solo para admins. Externos pueden consultar.
  • Productos: asociados a empresas. CRUD solo para admins.
  • Inventario: muestra cantidades por producto y empresa.
  • PDF: se puede descargar el inventario en formato PDF.
  • Correo: se simula el envío de PDF por correo.
  • IA: genera descripciones de producto automáticamente.

🧪 Ejemplo de Prueba IA

Endpoint:

POST /api/generar-descripcion/
{ "nombre": "Laptop Gamer", "caracteristicas": "Intel i9, 32GB RAM, RTX 4080" }

Respuesta esperada:

{ "descripcion_generada": "El producto 'Laptop Gamer' destaca por sus características..." }

🔗 Endpoints Importantes

Recurso Método URL
Login POST /api/login/
Empresas GET /api/empresas/
Productos GET /api/productos/
Generar descripción (IA) POST /api/generar-descripcion/
Inventario (PDF) GET /api/inventario/pdf/
Enviar PDF (correo) GET /api/inventario/enviar/
Presentación técnica GET /presentacion/

⚙️ Configuraciones Técnicas

  • Permisos personalizados por rol usando IsAdminOrReadOnly
  • Correo simulado con EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'
  • Modelo CustomUser como AUTH_USER_MODEL

📝 Notas Finales

  • El sistema es fácilmente extensible a PostgreSQL o servicios externos de correo.
  • La IA puede conectarse con OpenAI usando una API Key real.
  • Todo el sistema fue probado y documentado con ejemplos funcionales.
